# Kanban Projet — Template

Modèle de kanban pour un projet DOMORIA.

## Colonnes

1. **Backlog** — Tout ce qu'il y a à faire, sans priorisation
2. **À faire** — Tâches prêtes à être traitées (priorisées)
3. **En cours** — Quelqu'y travaille activement
4. **Review** — Fait, en attente de validation/revue
5. **Fait** — Terminé et validé

## Convention de nommage des cartes

- `[TÂCHE]` Description courte de la tâche
- `[BUG]` Description du bug
- `[FEATURE]` Description de la fonctionnalité
- `[ADMIN]` Tâche administrative / organisationnelle
- `[JALON]` Jalon / deadline importante

## Tags utilises

- `prioritaire` — À traiter en premier
- `blocking` — Bloque d'autres tâches
- `quick-win` — Rapide à faire, fort impact
- `external` — Dépend d'un tiers externe

## Fréquence de mise à jour

- Revue kanban : **chaque lundi** (ou plus si nécessaire)
- Les cartes "En cours" ne doivent pas rester plus de **3 jours** sans mise à jour
